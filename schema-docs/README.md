# Schema Docs (Legacy Supabase Migrations)

Esta carpeta contiene las migraciones SQL originales de cuando el proyecto usaba Supabase.

**El backend actual es Insforge.** Estos archivos sirven solo como documentacion/referencia del esquema de la base de datos.

Para aplicar cambios en la BD, usar `run-raw-sql` de Insforge directamente.

## Estructura

- `migrations/` — SQL files que documentan la evolucion del esquema
- `seed-dev.sql` — Datos de prueba para desarrollo
- `storage/` — Configuracion legacy de storage
